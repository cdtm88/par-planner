<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Internet → Worker | Public `POST /api/rooms`, `GET /api/rooms/*`, WS upgrade. All untrusted. |
| Worker → Durable Object | Trusted internal RPC over `stub.fetch`; no auth needed (binding is private). |
| WS connection → DO `onMessage` | Client payloads are hostile until `v.safeParse` succeeds. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-02-01 | Spoofing | WS `hello.playerId` | accept (Phase 1) | Client self-declares `playerId`. Phase 1 always admits — Phase 5 RESI-02 adds slot-hold matching. Server never grants privileges based on the claimed id alone. |
| T-01-02-02 | Tampering | WS payload | mitigate | `v.safeParse(ClientMessage, JSON.parse(raw))` in `party/game-room.ts onMessage`; malformed → `{type:"error",code:"bad_message"}`; connection stays open so attacker gets no oracle. |
| T-01-02-03 | Repudiation | Display-name abuse | partial-mitigate | Valibot caps length at 20; Svelte's default HTML escaping (Plan 03/04) prevents XSS. Profanity filtering is v2 (MODR-02). |
| T-01-02-04 | Information Disclosure | Room-code enumeration | mitigate | 31^6 ≈ 887M combinations. `/exists` and WS upgrade return identical 404 for "never existed" and "reaped" — no oracle. No rate limiting in Phase 1 (see T-01-02-07). |
| T-01-02-05 | Denial of Service | WS message flood | accept (Phase 1) | Hibernation absorbs idle cost; Cloudflare applies per-IP connection ceilings globally. In-DO rate-limiting is Phase 3 scope. |
| T-01-02-06 | Elevation of Privilege | Client-claimed "I am host" | mitigate | `party/game-room.ts` owns `#hostId`; it's set on the FIRST `hello` when `#players.size === 0 && #hostId === null`. The `hello` payload has NO `isHost` field in the schema — client cannot self-grant. |
| T-01-02-07 | DoS / Cost | `POST /api/rooms` spam | accept (Phase 1) | Documented risk. Cloudflare Workers have a per-zone rate limit (free tier 100k req/day). Phase 3 will add Turnstile or per-IP bucket per RESEARCH.md §Security Domain. |
| T-01-02-08 | Spoofing | Cross-origin WS hijack | mitigate | `onBeforeConnect` in `src/worker.ts` currently enforces existence; origin allowlist (`env.APP_ORIGIN` comparison) is added in a follow-up when we have a prod origin. Documented for Phase 5/6 hardening. |
| T-01-02-09 | Information Disclosure | Server-error stack traces | mitigate | `src/hooks.server.ts` `handleError` returns generic message, logs details server-side. ASVS V11. |
</threat_model>
