# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1 (Foundation — Transport, Room, Lobby, Presence). Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Tailwind v4 + `@theme` tokens, no component library) |
| Preset | not applicable (shadcn is React-only; stack is SvelteKit + Svelte 5 — see research/STACK.md §Styling) |
| Component library | none — small surface (home, modal, lobby, error page, banner) built as first-party Svelte components under `src/lib/components/` |
| Icon library | `lucide-svelte` ^0.454.0 — covers crown (host), clipboard/check (copy), loader (reconnecting), arrow-right (CTAs), alert-triangle (error). Ships tree-shakeable SVGs; matches the clean-but-playful voice. |
| Font | Primary UI: **Inter** (variable) via `@fontsource-variable/inter` — reads crisply at small sizes on mobile. Display/brand wordmark: **Space Grotesk** (variable) via `@fontsource-variable/space-grotesk` — gives the "Bullshit Bingo" title the slightly off-kilter personality the name demands without sacrificing legibility. Both self-hosted (no Google Fonts CDN round-trip on a Cloudflare-hosted app). |

Phase 1 surface scope: Home page, Display-name modal, Lobby screen, Error page (expired/unknown room), Reconnecting banner.

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps inside buttons, chip internal padding |
| sm | 8px | Compact element spacing (code digits, roster meta) |
| md | 16px | Default element spacing, button internal padding, modal form rows |
| lg | 24px | Section padding inside cards, list-item vertical rhythm in the roster |
| xl | 32px | Major layout gaps (between lobby header block and roster block) |
| 2xl | 48px | Vertical spacing between hero heading and CTA group on home |
| 3xl | 64px | Page-level top/bottom padding on desktop home hero |

**Exceptions:**
- **44px minimum tap target** for all interactive elements on mobile (per BOAR-07 which this phase pre-seeds; matches iOS HIG). Declared as a non-scale exception so `h-11 w-11` is the baseline for icon-only controls (copy buttons, modal close).
- **Container max-width: 480px** on home and modal, **640px** on lobby. Not a spacing token — a layout constraint. Pages are single-column, centered, mobile-first.

## Typography

Four sizes, two weights, two families. Line heights as ratios.

| Role | Size | Weight | Line Height | Family |
|------|------|--------|-------------|--------|
| Body | 16px | 400 (regular) | 1.5 | Inter |
| Label | 14px | 600 (semibold) | 1.4 | Inter |
| Heading | 24px | 600 (semibold) | 1.2 | Inter |
| Display | 40px mobile / 56px ≥640px | 600 (semibold) | 1.1 | Space Grotesk |

**Usage map:**
- **Body (16px/400/1.5 Inter):** paragraph copy, helper text under inputs, empty-state body, error page body, player names in roster.
- **Label (14px/600/1.4 Inter):** button text, input labels, the "Host" badge, "Room code" / "Share link" captions, reconnecting banner text.
- **Heading (24px/600/1.2 Inter):** lobby section heading ("Players · {n}"), error page heading, modal title.
- **Display (40/56px/600/1.1 Space Grotesk):** "Bullshit Bingo" wordmark on home, room code displayed in the lobby header (monospace-feeling via Space Grotesk's geometric forms; letter-spacing 0.1em on the code).

**Only two weights declared: 400 and 600.** No italic, no 500/700.

## Color

A single-accent playful palette. Dark-friendly surfaces with a vivid accent reserved for primary action + host identity. Values declared as CSS custom properties in `@theme`.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0F0F14` (near-black charcoal) | Page background, modal backdrop underlay |
| Secondary (30%) | `#1A1A23` (elevated surface) + `#2A2A36` (borders/dividers) | Cards, lobby header block, modal body, roster rows, input backgrounds |
| Accent (10%) | `#F5D547` (bingo-card yellow) | Primary CTA buttons, room code text, host-crown fill, the "BS Bingo" wordmark punctuation dot |
| Destructive | `#F87171` (warm red-coral) | Error page icon + headline accent, "Leave / cancel" in modal if present |

**Ink / text colors** (derived from surfaces, not a separate palette):
- Primary text: `#F5F5F7` on dominant/secondary surfaces
- Secondary text: `#A1A1AA` (hints, "waiting for players…", helper text)
- Disabled text: `#52525B` (disabled "Start Game" button label)
- Primary CTA text on accent: `#0F0F14` (near-black on yellow)

**Accent reserved for:**
1. Primary CTA button fill ("Create a game", "Join", "Copy code", "Copy link")
2. Room code display text in the lobby header
3. Host crown icon fill
4. The decorative dot after the "Bullshit Bingo" wordmark on the home hero (one pixel of personality)

Accent is NOT used for: input focus rings, hover states on neutral buttons, links, or roster highlights. Focus rings use a 2px `#A1A1AA` outline offset 2px. Secondary buttons ("Join a game" when it is the counterpart to the primary "Create a game", and "Copy link" when paired with "Copy code") use a `#2A2A36` outline on `#1A1A23` — accent-free.

**Contrast audit (WCAG AA floors):**
- Primary text `#F5F5F7` on `#0F0F14`: 17.1:1 — passes AAA
- Primary text `#F5F5F7` on `#1A1A23`: 15.3:1 — passes AAA
- Secondary text `#A1A1AA` on `#0F0F14`: 8.4:1 — passes AAA
- CTA text `#0F0F14` on accent `#F5D547`: 14.1:1 — passes AAA
- Destructive `#F87171` on `#0F0F14`: 7.2:1 — passes AAA

## Copywriting Contract

Voice: **fun, slightly irreverent, not corporate.** Short sentences. Second person. Never announce the app state to the user as if they are a screen reader ("You have successfully joined the lobby"). Say what they can do next.

| Element | Copy |
|---------|------|
| Home — brand wordmark | `Bullshit Bingo` |
| Home — tagline | `The meeting game. Mark the buzzwords, first to a line wins.` |
| Home — primary CTA | `Create a game` |
| Home — secondary CTA (inline) | `Join with code` (label above a 6-char input and a `Join` submit) |
| Display name modal — title | `What should we call you?` |
| Display name modal — input placeholder | `Your name` |
| Display name modal — input helper | `Max 20 characters. Nothing permanent — just for this game.` |
| Display name modal — submit (create flow) | `Create game →` |
| Display name modal — submit (join flow) | `Join game →` |
| Display name modal — empty validation | `Pick a name first.` |
| Lobby — header label | `Room code` |
| Lobby — copy-code button label | `Copy code` (becomes `Copied` with check icon for 2s on success) |
| Lobby — share link label | `Share link` |
| Lobby — copy-link button label | `Copy link` (becomes `Copied` with check icon for 2s on success) |
| Lobby — roster heading | `Players · {n}` |
| Lobby — host badge text | `Host` (with crown icon) |
| Lobby — "waiting" hint (n<2) | `Waiting for players. Share the code or link to get going.` |
| Lobby — start game button (host, disabled in Phase 1) | `Start Game` (disabled with tooltip/helper: `Coming in the next build — add words first.`) |
| Lobby — non-host hint | `Waiting for the host to start.` |
| Reconnecting banner | `Reconnecting…` (with spinner icon, left-aligned, non-blocking) |
| Error page — heading | `Room not found` |
| Error page — body | `That game is over, the code is wrong, or the room expired. Want to kick off a new one?` |
| Error page — CTA | `Create a new game` |

**Destructive actions in Phase 1:** None. Phase 1 has no "delete room", "kick player", or "leave game" flows. The display-name modal has an implicit dismiss (Esc / backdrop click) that returns the user to home — this is not destructive (no state exists yet to destroy), so no confirmation.

**Empty state:** The lobby itself is the empty state while n<2. Covered by the "Waiting for players" hint above — no separate empty illustration needed for this phase.

**Error state taxonomy:**
- **Room not found / expired** → full-page error (`Error page` rows above). Routed from `/join/{badCode}` and `/room/{deadId}`.
- **WebSocket disconnected** → non-blocking top banner (`Reconnecting…` row above). Never a full-page state in Phase 1; Phase 5 owns the deeper reconnect UX.
- **Invalid join code format** (on home) → inline helper text under the input: `Codes are 6 letters and numbers. Try again.` — does not block the rest of the page.
- **Empty display name** → inline under modal input, per copy above.

## Interaction Contracts

States for every interactive element. Executor must implement all four for each interactive element.

| Element | Default | Hover | Active / Pressed | Focus-visible | Disabled |
|---------|---------|-------|------------------|---------------|----------|
| Primary CTA (accent fill) | `bg-accent text-ink-inverse` | 8% lighter accent | translateY(1px), no shadow | 2px outline `#A1A1AA` offset 2px | opacity 40%, cursor not-allowed |
| Secondary CTA (outlined) | `bg-secondary border border-divider` | border brightens to `#3A3A48` | translateY(1px) | 2px outline `#A1A1AA` offset 2px | opacity 40% |
| Icon button (copy) | `text-ink-primary bg-secondary` | `bg-#2A2A36` | scale 0.97 | 2px outline | n/a |
| Text input | `bg-secondary border border-divider` | border `#3A3A48` | n/a | border `#A1A1AA` 2px | n/a |

**Copy feedback:** On successful Clipboard API write, the button label swaps to `Copied` with a `lucide-check` icon for 2000ms, then reverts. No toast. No sound. The accent color stays on the primary button; this is a label change only.

**Modal behavior:** Centered, max-width 480px, backdrop `#0F0F14/80` blur-none. Esc and backdrop click dismiss. First input receives autofocus on open. Submit on Enter inside the input. Tab trap within modal until dismissed.

**Reconnecting banner:** Position fixed, top 0, full width, 8px vertical padding, `#1A1A23` background with a 1px bottom border `#2A2A36`. Enters with a 150ms translateY(-100%)→0 transition. Auto-dismisses on reconnect with the reverse transition. Never blocks input on the page behind it.

**Join code input:** 6 monospace characters, each letter auto-uppercases. Alphabet `ABCDEFGHJKMNPQRSTUVWXYZ23456789` (per CONTEXT D-05). Paste handler strips whitespace and ambiguous chars, not an inline re-map error — just silently drop. Submit enabled when exactly 6 valid chars present.

## Responsive Behavior

Mobile-first. Breakpoints use Tailwind v4 defaults:

| Breakpoint | Width | Changes |
|------------|-------|---------|
| base | <640px | Single column, page padding 16px (md), Display size 40px, CTAs full-width |
| sm | ≥640px | Display size 56px, page padding 32px (xl), CTAs intrinsic width (min 180px) |
| md | ≥768px | Lobby container max-width 640px, header lays code + share-link side-by-side; below this they stack |
| lg | ≥1024px | Home hero gains 64px (3xl) top padding; no other layout change |

All tap targets on mobile are ≥44px (min-h-11 / min-w-11). Forms are always single-column — no multi-column forms in this phase.

## Motion & Animation

Restrained and functional. Nothing bouncy in Phase 1 (celebration is Phase 4's job).

| Moment | Motion |
|--------|--------|
| Player joins roster | Row fades in over 120ms, no slide |
| Player leaves roster | Row fades out over 120ms |
| Copy feedback | Label cross-fades (100ms) between `Copy code` ↔ `Copied` |
| Modal open | Backdrop fade 120ms; sheet scale 0.98→1 + fade 150ms |
| Modal close | Reverse, 100ms |
| Reconnecting banner | translateY 150ms ease-out on enter / ease-in on exit |
| Button press | 80ms translateY(1px); no spring |

Respect `prefers-reduced-motion: reduce` — collapse all transitions to 0ms except the copy label swap (which becomes instant).
