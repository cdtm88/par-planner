# Phase 3 — UI Design Contract

> Visual and interaction contract for Phase 3 (Board Generation & Core Mark Loop). Generated by gsd-ui-researcher. Inherits Phase 1/2 design system; extends it with board, cell, and per-player mark-count affordances.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Tailwind v4 + `@theme` tokens, no component library) |
| Preset | not applicable (shadcn is React-only; stack is SvelteKit + Svelte 5) |
| Component library | First-party Svelte components under `src/lib/components/` — Phase 3 adds `Board`, `BoardCell` |
| Icon library | `lucide-svelte` ^0.454.0 — no new icons required for Phase 3 (marked-cell state uses color contrast alone per D-13; no icon overlay) |
| Font | Inter Variable (body/label/heading) + Space Grotesk Variable (display) — unchanged |

Source: Phase 1 UI-SPEC, Phase 2 UI-SPEC, `src/app.css` @theme tokens (verified 2026-04-17), Phase 3 CONTEXT.md D-12 / D-13 / D-14.

Phase 3 surface scope: `<Board />` component rendered inside existing `/room/[code]` route when `state.phase === 'playing'` (D-01, D-02). The same page continues to show the player roster (with new mark-count badges per D-07) above or alongside the board.

## Spacing Scale

Unchanged from Phase 1/2. One clarification for the board surface.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, mark-count badge internal padding |
| sm | 8px | Cell-to-cell gap in the bingo grid, roster row internal gap, badge margin from name |
| md | 16px | Default section padding, gap between player-list block and board block |
| lg | 24px | Board container outer padding on ≥640px |
| xl | 32px | Not used in Phase 3 |
| 2xl | 48px | Not used in Phase 3 |
| 3xl | 64px | Not used in Phase 3 |

**Exceptions:**
- **44px minimum tap target** on every interactive word cell (BOAR-07, D-15). Non-negotiable — grid sizing must guarantee this at the narrowest supported viewport (320px).
- **Grid gap: 8px (sm token)** between cells at all grid tiers (3×3, 4×4, 5×5). Uses CSS `gap` property, not margins.
- **Container max-width: 640px** on `/room/[code]` (inherited from Phase 1). Board fills this container on mobile and capped width on desktop.

## Typography

Unchanged from Phase 1/2. No new sizes, weights, or families.

| Role | Size | Weight | Line Height | Family |
|------|------|--------|-------------|--------|
| Body | 16px | 400 (regular) | 1.5 | Inter |
| Label | 14px | 600 (semibold) | 1.4 | Inter |
| Heading | 24px | 600 (semibold) | 1.2 | Inter |
| Display | 40px mobile / 56px ≥640px | 600 (semibold) | 1.1 | Space Grotesk |

**Phase 3 usage map additions:**
- **Label (14px/600/1.4 Inter):** mark-count badge numeric text, cell word text at 3×3 and 4×4 tiers.
- **Body (16px/400/1.5 Inter):** not used inside cells (too tall for 44px minimum cells at 5×5); used for roster names and above-board section hints.
- **Cell word text sizing rule:** Phase 3 uses **Label 14px / 600** as the cell word size at every grid tier. At 5×5 on the narrowest viewport, long words (e.g. `Synergize`) are handled by a **two-line wrap with `hyphens: auto`** and `word-break: break-word` — no ellipsis truncation (truncating a word the player needs to recognize would defeat the purpose). Cells that must wrap render text centered, line-height 1.2, with padding reduced to 6px horizontal at 5×5 only.

**Only two weights declared: 400 and 600.** No italic, no 500/700. Unchanged from Phase 1.

## Color

All tokens from Phase 1/2. Zero additions. Phase 3 reuses every token in board-specific contexts, and the accent has one new, load-bearing responsibility: **the marked-cell fill.**

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0F0F14` | Page background (unchanged) |
| Secondary (30%) | `#1A1A23` (surface) + `#2A2A36` (dividers/borders) | Unmarked cell background, blank cell background, roster row background |
| Accent (10%) | `#F5D547` | **Marked cell fill** (D-13), mark-count badge fill when count > 0, roster row glow when a player just marked (ephemeral, see Motion) |
| Destructive | `#F87171` | Not used in Phase 3 (no destructive actions this phase) |

**Accent reserved for (Phase 3 additions, cumulative list):**
1. Primary CTA button fill (Phase 1 — `Create a game`, `Join`, copy buttons)
2. Room code display text in the lobby header (Phase 1)
3. Host crown icon fill (Phase 1)
4. Wordmark decorative dot (Phase 1)
5. Start Game button fill (Phase 2 — host, enabled)
6. Grid-progress bar fill (Phase 2)
7. **Marked cell fill on the player's own board (Phase 3 — D-13)** — `#F5D547` background, `#0F0F14` text
8. **Mark-count badge fill on PlayerRow when count > 0 (Phase 3 — D-07)** — `#F5D547` background, `#0F0F14` numeric text

**Accent is NOT used for:**
- Unmarked word cells (those stay `#1A1A23` surface per D-12)
- Blank/inert cells (those stay `#1A1A23` surface per D-14)
- Hover states on cells (hover brightens the 1px border to `#3A3A48`, never the fill)
- Focus rings (focus uses 2px `#A1A1AA` outline offset 2px — unchanged Phase 1 rule)
- Mark-count badge when count === 0 (badge is hidden entirely below 1; see below)
- Peer players' cells (peers never see another player's board layout — D-04)

**Phase 3 cell color contract:**

| Cell state | Background | Text | Border | Cursor |
|-----------|------------|------|--------|--------|
| Unmarked word cell (default) | `#1A1A23` | `#F5F5F7` | 1px `#2A2A36` | pointer |
| Unmarked word cell (hover) | `#1A1A23` | `#F5F5F7` | 1px `#3A3A48` | pointer |
| Unmarked word cell (active/pressed) | `#1A1A23` | `#F5F5F7` | 1px `#3A3A48`, scale 0.97 | pointer |
| Unmarked word cell (focus-visible) | `#1A1A23` | `#F5F5F7` | 2px `#A1A1AA` outline, offset 2px | pointer |
| **Marked word cell** | `#F5D547` (accent) | `#0F0F14` (ink-inverse) | 1px `#F5D547` | pointer |
| Marked word cell (hover) | `#F5D547` | `#0F0F14` | 1px `#F5D547` | pointer |
| Marked word cell (focus-visible) | `#F5D547` | `#0F0F14` | 2px `#A1A1AA` outline, offset 2px | pointer |
| **Blank/inert cell** | `#1A1A23` | — (no text) | 1px dashed `#2A2A36` at 40% opacity | default |
| Blank cell (any hover/focus) | unchanged — no hover/focus behavior (non-interactive per D-11) | — | unchanged | default |

**Blank cell visual language (D-09, D-11, D-14):** Same surface as unmarked cells (`#1A1A23`) so they sit within the grid without looking broken, but the dashed border at 40% opacity clearly signals "not a real cell." No text. No cursor-pointer — the default cursor. No hover transition. `aria-hidden="true"`. `tabindex="-1"`. Cannot receive focus via keyboard navigation.

**Mark-count badge contract (D-07):**

| Badge state | Background | Text | Position |
|------------|------------|------|----------|
| Count === 0 | hidden (not rendered) | — | — |
| Count ≥ 1 | `#F5D547` | `#0F0F14` (ink-inverse) | right-aligned in PlayerRow, 8px (sm) from name |
| Count for current viewer | `#F5D547` | `#0F0F14` | same — no distinction from peers' badges |

Badge renders as a 20px-tall pill with 8px horizontal padding, Label 14px/600 numeric text.

**Contrast audit (Phase 3 additions, WCAG AA floors):**
- Marked cell: `#0F0F14` on `#F5D547` → 14.1:1 — passes AAA (inherited from Phase 1 primary CTA)
- Unmarked cell: `#F5F5F7` on `#1A1A23` → 15.3:1 — passes AAA
- Mark-count badge: `#0F0F14` on `#F5D547` → 14.1:1 — passes AAA
- Blank cell border: dashed `#2A2A36` at 40% (effective ≈ `#1A1A22`) on `#1A1A23` — non-text, informational only; visual-only signal is reinforced by absence of text, absence of cursor-pointer, and `aria-hidden`.

## Copywriting Contract

Voice inherited: fun, slightly irreverent, not corporate. Second person. Short sentences.

Phase 3 is an almost copy-free screen — the board is its own UI. Below is every string that does render.

| Element | Copy |
|---------|------|
| Board section heading (above the grid, label 14px secondary ink) | `Your board` |
| Empty-state heading (should be unreachable in Phase 3, but render defensively) | `Dealing your board…` |
| Empty-state body (defensive, only visible if `boardAssigned` hasn't arrived yet) | `Hang tight — we're shuffling the words.` |
| Mark-count badge aria-label | `{n} marks` (e.g. `3 marks`) |
| Word cell aria-label (unmarked) | `{word}. Tap to mark.` |
| Word cell aria-label (marked) | `{word}. Marked. Tap to unmark.` |
| Blank cell aria-label | none — `aria-hidden="true"` (D-11) |
| PlayerRow mark-count aria annotation | appended to existing row label: `{PlayerName}, {n} marks` (singular `1 mark`) |
| Non-host / peer hint (above roster or below board — optional, only if space permits on desktop) | `Mark the buzzwords as you hear them. First to a full line wins.` (Body 16px, `#A1A1AA` secondary ink) |
| Error: `boardAssigned` never arrives within 3s of `gameStarted` | `Couldn't deal your board. Try refreshing the page.` (Body 16px, `#F87171`) — defensive only; server-side this should never happen |
| Error: `markWord` rejected by server (e.g. invalid cellId) | silent — optimistic toggle reverts, no visible error. Reason: Phase 3 is happy-path-dominant; rejections are only possible through client bugs, not user error. |

**Destructive actions in Phase 3:** **None.** Tapping a marked cell unmarks it — this is a toggle, not destruction. No confirmation. No undo prompt. The cell simply returns to unmarked.

**Empty state:** The `Dealing your board…` + `Hang tight — we're shuffling the words.` pair covers the theoretical gap between `gameStarted` broadcast and the per-connection `boardAssigned` arrival (D-03 specifies this ordering). In practice on Cloudflare DO this gap is <50ms and the state is never visibly rendered, but the copy exists so the executor has a defined fallback rather than a blank grid.

**Error state taxonomy:**
- **Board never arrives (3s timeout):** full-board-area error with refresh copy above. Not a full-page error — the roster and room code remain visible.
- **Mark rejected by server:** silent optimistic rollback. No toast, no banner. (Rejection is a client-bug signal, not a user-addressable error.)
- **Disconnect during play:** inherited Phase 1 `Reconnecting…` banner. Phase 5 owns deeper reconnect UX; Phase 3 leaves the board visible (stale marks OK) under the reconnecting banner.

## Interaction Contracts

Inherits Phase 1 (Button, TextInput, Modal, Banner) and Phase 2 (WordChip, WordPool, PackPills, GridProgress) interaction states. Phase 3 additions:

| Element | Default | Hover | Active / Pressed | Focus-visible | Disabled |
|---------|---------|-------|------------------|---------------|----------|
| Word cell (unmarked) | `bg-surface border-divider text-ink-primary` | border brightens to `#3A3A48` | scale 0.97, 60ms | 2px outline `#A1A1AA` offset 2px | n/a (cells don't disable in Phase 3; Phase 4 may freeze after win) |
| Word cell (marked) | `bg-accent text-ink-inverse border-accent` | no fill change | scale 0.97, 60ms | 2px outline `#A1A1AA` offset 2px | n/a |
| Blank cell | `bg-surface border-dashed-divider-40 text-transparent` | no change | no change | not focusable (tabindex -1) | n/a (never interactive) |
| Mark-count badge | `bg-accent text-ink-inverse` (count ≥ 1); hidden (count === 0) | no hover (badge is display-only, not interactive) | n/a | n/a | n/a |

**Optimistic mark toggle (D-06):**
On cell tap:
1. Client immediately toggles the cell's local `marked` state and sends `markWord { cellId }` to the server.
2. Server confirms via `wordMarked` broadcast (which contains the actor's `playerId` and new `markCount`, not the cell ID — layout stays private per D-04).
3. If the server response disagrees with the optimistic state (should not happen in Phase 3; can only occur on client bug), revert silently.

This means there is no loading spinner on tap. The visual transition is instant. The only observable round-trip is peers' mark-count badge updating ~1s later on their screens (per BOAR-06 / success criterion #4).

**Keyboard interaction:**
- Cells are `<button>` elements (real buttons, not divs). Native Tab navigation moves between word cells in DOM order (row-major, left-to-right, top-to-bottom).
- Blank cells are `tabindex="-1"` and skipped by Tab.
- Space or Enter on a focused cell toggles its marked state.
- Focus ring is the Phase 1 standard: 2px `#A1A1AA` outline offset 2px.

**Mark-count badge update:**
When `wordMarked` arrives, the affected PlayerRow's badge updates its numeric text. There is no "flash" by default (avoids noise in a 5-player game where marks happen often); see Motion for the optional ephemeral glow.

## Responsive Behavior

Inherits Phase 1/2 breakpoints. Phase 3 adds the board grid sizing contract.

| Breakpoint | Width | Board behavior |
|------------|-------|----------------|
| base | <640px (mobile portrait) | Grid fills container width. Container has 16px page padding. 8px gap between cells. Cells are square (aspect-ratio 1). Text wraps at 5×5, single line at 3×3 and 4×4. |
| sm | ≥640px | Container max-width 640px (inherited). Board uses full container width. Cells remain square. |
| md | ≥768px | No layout change for the board itself. On desktop, the roster may sit to the left of the board in a two-column layout (Claude's Discretion: for Phase 3, keep the single-column stack — roster above, board below — to avoid reflow. Revisit in Phase 4 if celebration screen needs two-column.) |

**Cell sizing at each tier (320px-wide worst case, 16px container padding, 8px gaps):**

- **3×3 tier** (9 cells, 2 gaps per row): cell = (320 − 32 − 16) / 3 = **~90px square.** Tap target comfortably exceeds 44px. Text `Label 14px` fits on one line for almost all words.
- **4×4 tier** (16 cells, 3 gaps per row): cell = (320 − 32 − 24) / 4 = **~66px square.** Still exceeds 44px. Text wraps to two lines for 8+ character words.
- **5×5 tier** (25 cells, 4 gaps per row): cell = (320 − 32 − 32) / 5 = **~51px square.** Meets 44px minimum. Padding reduced to 6px horizontal at this tier (vs 8px at others). Text wraps to two lines routinely; `hyphens: auto` smooths multi-syllable words.

**Viewport overflow protection:** The board container uses `max-width: 100%` and cells use `min-width: 44px`. If a viewport is narrower than `3 × 44 + 2 × 8 = 148px` (never realistic — iPhone SE is 375px), the grid scrolls horizontally as a last resort rather than shrinking tap targets below 44px.

**Orientation:** Phase 3 is optimized for portrait (meeting-in-browser on a phone). Landscape works — the grid still fills container width — but the roster may need to collapse above the board on landscape mobile. Claude's Discretion: for Phase 3 keep the roster as a compact strip on landscape mobile (just initials + mark count, no name expansion); full spec in Phase 5.

## Motion & Animation

Extends Phase 1/2 motion contract (restrained, functional, celebration is Phase 4's job).

| Moment | Motion |
|--------|--------|
| Cell tap (mark) — own board | Fill color transitions `#1A1A23 → #F5D547` over 120ms ease-out. Text color transitions `#F5F5F7 → #0F0F14` in the same window. Border transitions in parallel. No scale "pop" — tap register is immediate and the scale 0.97 active-state covers the haptic moment. |
| Cell tap (unmark) — own board | Reverse of above, 120ms ease-out. |
| Cell active/pressed | scale 0.97 over 60ms (matches chip × delete in Phase 2 for consistency) |
| `boardAssigned` arrival — cells fade in | Cells fade in (opacity 0 → 1) with a 20ms stagger per row over 200ms total. Stagger is per-row, not per-cell, to stay subtle. |
| `wordMarked` arrives from a peer — PlayerRow badge | Badge numeric text updates in place (no animation). Optionally the PlayerRow gets a 300ms accent-colored box-shadow pulse (4px blur, `#F5D547` at 20% opacity) to signal "someone just marked." Claude's Discretion: enable the pulse in Phase 3; it adds "live meeting" energy without noise. Collapsed under prefers-reduced-motion. |
| Own mark-count badge update | Same behavior as peer — numeric text updates in place + optional 300ms pulse on own PlayerRow. |
| Layout shift on `phase: lobby → playing` | Crossfade the lobby content out (120ms) and the board in (120ms). No slide. Roster stays put; only the below-roster region swaps. |

**Respect `prefers-reduced-motion: reduce`:**
- Cell mark/unmark color transitions collapse to 0ms (instant color change).
- Cell scale active-state collapses to 0ms.
- `boardAssigned` stagger collapses to 0ms — all cells appear simultaneously.
- PlayerRow pulse collapses to no pulse (no box-shadow at all).
- Phase transition crossfade collapses to instant swap.

All Phase 1/2 motion rules remain in force.

## Component Inventory (Phase 3)

New first-party Svelte components. All added under `src/lib/components/`.

| Component | File | Purpose |
|-----------|------|---------|
| `Board` | `Board.svelte` | Root board container. Props: `cells: BoardCell[]`, `markedCellIds: Set<string>`, `onToggleMark: (cellId: string) => void`. Derives grid-tier (3/4/5) from `cells.length` (9/16/25). Renders a CSS grid with `grid-template-columns: repeat(N, 1fr)` and `aspect-ratio: 1/1` per cell. Delegates cell rendering to `BoardCell`. Handles the `boardAssigned` empty-state fallback. |
| `BoardCell` | `BoardCell.svelte` | Single cell. Props: `cell: BoardCell` (`{ cellId, wordId, text, blank }`), `marked: boolean`, `onToggle?: () => void`. If `blank === true`, renders non-interactive div with dashed border. Else renders a `<button>` with appropriate marked/unmarked styling and aria-label. |

**Modified Phase 1/2 components:**
- `PlayerRow.svelte` — add optional mark-count badge prop: `markCount?: number`. Renders the badge (accent pill with numeric text) when `markCount > 0`. Extends the existing row aria-label (D-07). No breaking change — the prop is optional so lobby usage in Phases 1/2 is unaffected.
- No other Phase 1/2 components change.

**No new icons** — marked state relies on color contrast alone (D-13). No `lucide-check` or `lucide-circle` overlay. Accessibility is covered by aria-label updates on the cell button, not by an icon.

**Shared types (add to `src/lib/protocol/messages.ts` per CONTEXT code-context):**
```ts
// BoardCell — the per-cell payload sent in `boardAssigned`
type BoardCell = {
  cellId: string      // nanoid, stable for mark tracking
  wordId: string | null   // null iff blank === true
  text: string | null     // null iff blank === true
  blank: boolean
}
```
