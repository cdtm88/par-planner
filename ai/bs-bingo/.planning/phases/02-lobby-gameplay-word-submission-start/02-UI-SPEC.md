## Copywriting Contract

Voice inherited: fun, slightly irreverent, not corporate. Second person. Short sentences.

| Element | Copy |
|---------|------|
| Word pool section heading | `Words (n)` where n is live count — source: CONTEXT.md D-03 |
| Word pool empty state heading | `No words yet` |
| Word pool empty state body | `Add buzzwords you expect to hear. The pool grows as everyone contributes.` |
| Word submission input placeholder | `Add a buzzword…` |
| Word submission Add button | `Add` |
| Duplicate rejection inline error | `"{word}" is already in the pool.` (shakes input, clears on next keystroke) — source: CONTEXT.md D-06 |
| Max-length validation | `30 characters max.` |
| Starter pack section label (host-only) | `Seed from a starter pack:` — source: CONTEXT.md D-07 |
| Pack pill labels | `Corporate Classics`, `Agile`, `Sales` — source: CONTEXT.md D-07 |
| Pack used tooltip / aria-label | `Already loaded` |
| Grid progress hint — below minimum | `Need {n} more word{s} to start a {size} board` |
| Grid progress hint — minimum met | `Ready for a {size} board — host can start!` (non-host) / `Ready — start when you are.` (host, inline below button) |
| Start Game button (host, enabled) | `Start Game` |
| Start Game button (host, disabled) | `Start Game` (button disabled; threshold hint below it explains why) |
| Non-host — waiting for host | `Waiting for {HostName} to start the game…` — source: CONTEXT.md D-13 |
| Word chip × button aria-label | `Remove "{word}"` |
| Destructive: remove own word | No confirmation dialog. Direct tap-to-remove. The chip disappears and the pool updates. Undo is out of scope. |

**Destructive actions:** Removing a word (LOBB-03) is the only destructive action. It is low-stakes (pool is collaborative, still in lobby), so no confirmation is required — tap × → word removed immediately, pool updates in real time for all players.

**Error states:**
- Duplicate word: inline under input, `#F87171`, clears on next keystroke. Not a toast.
- Max length exceeded: input caps at 30 chars natively (`maxlength`); no error state needed — the character limit is self-enforcing.
- Start Game while below minimum: button is disabled; threshold hint copy is the error explanation. No separate error shown.

## Interaction Contracts

Inherits all Phase 1 interaction states. Phase 2 additions:

| Element | Default | Hover | Active / Pressed | Focus-visible | Disabled |
|---------|---------|-------|------------------|---------------|----------|
| Word chip (own) | `bg-surface border-divider text-ink-primary` | border brightens to `#3A3A48` | no translateY (chip is static) | 2px outline `#A1A1AA` offset 2px | n/a |
| Chip × delete button | `text-ink-secondary` | `text-ink-primary` | scale 0.92, 60ms | 2px outline `#A1A1AA` offset 2px | n/a |
| Word chip (others') | `bg-surface border-divider text-ink-primary` | no hover change (non-interactive) | n/a | n/a | n/a |
| Pack pill (unused) | `bg-surface border-divider text-ink-primary` | border `#3A3A48` | translateY(1px), 80ms | 2px outline `#A1A1AA` offset 2px | n/a |
| Pack pill (used) | `bg-surface border-divider text-ink-disabled` | no change | no change | 2px outline `#A1A1AA` offset 2px | cursor not-allowed |
| Start Game (enabled) | `bg-accent text-ink-inverse` | 8% lighter accent | translateY(1px), 80ms | 2px outline `#A1A1AA` offset 2px | n/a |
| Start Game (disabled) | `bg-accent/40 text-ink-inverse/40` | no change | no change | 2px outline `#A1A1AA` offset 2px | cursor not-allowed |
| Word submission input | inherited TextInput default | border `#3A3A48` | n/a | border `#A1A1AA` 2px | n/a |
| Word submission Add button | inherited Button primary | inherited | inherited | inherited | opacity 40% (empty input) |

**Input shake animation (duplicate rejection):** `transform: translateX` oscillates ±6px over 300ms (3 cycles, ease-in-out). CSS keyframe, respect `prefers-reduced-motion: reduce` (collapse to 0ms — just show error text without shake).

**Pack loading behavior:** When a pack pill is tapped, words merge into the pool immediately — all chips appear simultaneously (not sequentially) via the standard chip fade-in. The pill transitions to used state in the same render. No loading spinner needed — the operation is fast and server-authoritative.

**Word pool scroll behavior:** The page scrolls as one unit. The word pool does not have an independent scroll container. On mobile, the user scrolls the page to reach the input and Start Game controls.

## Responsive Behavior

Inherits Phase 1 breakpoints. Phase 2 additions:

| Breakpoint | Width | Changes |
|------------|-------|---------|
| base | <640px | Word chips wrap freely, full-width input + Add button stack (input above, Add below, both full-width) |
| sm | ≥640px | Input and Add button inline (input flex-1, Add intrinsic min-width 80px) |
| md | ≥768px | Lobby container max-width 640px (unchanged) |

**Word chip sizing:** `py-2 px-3` (8px vertical, 12px horizontal). Min-height 36px at rest. The × delete tap target is 44×44px via negative margin / padding expansion so the visual chip can remain compact.

**Pack pills:** Wrap freely on mobile. No horizontal scroll. Each pill min-height 44px.

**Progress bar:** Full-width within the lobby container. Tier markers (`3×3`, `4×4`, `5×5`) rendered as absolute-positioned labels above the bar at the correct proportional positions.

## Motion & Animation

Extends Phase 1 motion contract (restrained and functional).

| Moment | Motion |
|--------|--------|
| Word chip added to pool | Fade in 120ms, no slide (consistent with player-joins-roster in Phase 1) |
| Word chip removed from pool | Fade out 120ms |
| Duplicate rejection shake | ±6px translateX oscillation, 300ms, 3 cycles — collapses to 0ms under prefers-reduced-motion |
| Pack pill → used state | Instant (same frame as pool update; no separate transition needed) |
| Progress bar fill change | Width transition 200ms ease-out |
| Start Game button enable/disable | Opacity transition 120ms (accent to accent/40) |
| Chip × delete button | scale 0.92 over 60ms on press |

All Phase 1 motion rules (button press translateY 80ms, prefers-reduced-motion collapse) remain in force.

## Component Inventory (Phase 2)

New first-party Svelte components. All added under `src/lib/components/`.

| Component | File | Purpose |
|-----------|------|---------|
| `WordChip` | `WordChip.svelte` | Renders a word from the pool. Props: `word: string`, `canDelete: boolean`, `onDelete?: () => void`. Deletable variant shows `lucide-x` with 44px tap target. Read-only variant text-only. |
| `WordPool` | `WordPool.svelte` | Wrapping chip container with section header "Words (n)". Props: `words: WordEntry[]`, `playerId: string`. Renders empty state when words.length === 0. Delegates chip rendering to `WordChip`. |
| `PackPills` | `PackPills.svelte` | Host-only section. Three pill buttons. Props: `usedPacks: Set<string>`, `onLoad: (pack: string) => void`. Used state shows `lucide-check` + greyed styling. |
| `GridProgress` | `GridProgress.svelte` | Progress bar + tier markers + threshold hint copy. Props: `wordCount: number`. Derives current tier and next threshold. Renders hint copy and progress fill. |

**Modified Phase 1 components:**
- `TextInput.svelte` — add shake-animation CSS class trigger via a reactive prop (`shake: boolean`). Existing error display pattern already handles the inline error text.
- `Button.svelte` — no changes needed; existing `primary` + disabled variants cover Start Game.
- `Badge.svelte` — no changes needed for Phase 2.

## Screen-by-Screen Additions (Lobby — Phase 2 Layer)

The lobby page (`src/routes/room/[code]/+page.svelte`) gains these sections below the roster block:

**Word Pool section (all players)**
- [ ] Section header `Words (n)` (Heading 24px), live-updating count
- [ ] `WordPool` component with chips in wrapping flow layout
- [ ] Empty state: `No words yet` + body copy when words.length === 0
- [ ] Own words show × delete button (min 44px tap target)
- [ ] Others' words show text only — no delete affordance

**Word submission (all players)**
- [ ] `TextInput` with placeholder `Add a buzzword…`, maxlength 30
- [ ] `Add` button (primary, right of input on ≥640px, stacked below on mobile)
- [ ] Submit on Enter key or Add tap
- [ ] On duplicate: input shakes + `"X" is already in the pool.` error below input
- [ ] Error clears on next keystroke

**Starter pack section (host only — not rendered for non-hosts)**
- [ ] Label `Seed from a starter pack:` (Label 14px, secondary ink)
- [ ] Three pack pills: `Corporate Classics`, `Agile`, `Sales`
- [ ] Used pills: greyed + check icon, non-interactive, aria-label `Already loaded`

**Grid progress + Start Game (host)**
- [ ] `GridProgress` component showing word count toward next tier
- [ ] Tier markers at 3×3 (5 words), 4×4 (12 words), 5×5 (21 words) positions
- [ ] Threshold hint copy below bar (see copywriting contract)
- [ ] `Start Game` button (accent, full-width mobile, min-width 180px ≥640px)
- [ ] Disabled state when wordCount < 5; enabled from 5 words onward
- [ ] Disabled start-game shows `Need {n} more word{s} to start a {size} board` hint below

**Grid progress + waiting (non-host)**
- [ ] Same `GridProgress` component (read-only, no Start Game button)
- [ ] `Waiting for {HostName} to start the game…` (Body 16px, `#A1A1AA`) in place of Start Game
